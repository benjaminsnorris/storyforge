# Structural Revise Mode (`--structural`)

## Summary

Add a `--structural` flag to `storyforge-revise` that reads structural proposals from validation, auto-generates a CSV-only revision plan, and executes fixes without touching prose. Mirrors the `--polish` pattern for the opposite end of the pipeline.

## Motivation

Structural scoring (`storyforge-validate --structural`) identifies novel-wide issues in the CSV data: thematic fragmentation, MICE thread health, pacing shape, arc completeness, character presence, scene function variety. These are all fixable at the CSV level — no prose changes needed. Currently there's no dedicated path for this; the revise skill handles it interactively but there's no autonomous mode that targets only structural improvements.

## Design

### Flag & Plan Generation

`storyforge-revise --structural` reads `working/scores/structural-proposals.csv`, generated by `storyforge-validate --structural`. If the file doesn't exist or has no pending proposals, exit with an error directing the user to run validation first.

For each pending proposal, generate a pass in `working/plans/revision-plan.csv`:

- **Group by dimension** — one pass per dimension, not per proposal. Multiple proposals for the same dimension (e.g., two arc_completeness findings) become one pass with combined targets.
- **fix_location** from the proposal (intent, structural, or registry).
- **targets** from the proposal's target field (scene IDs joined by `;`, or empty for global).
- **model_tier** set to `sonnet` (analytical work, not creative).
- **purpose** concatenated from proposal rationales.
- **status** set to `pending`.

Pass ordering follows existing principles: structural (scenes.csv) first, then intent (scene-intent.csv), then registry (values.csv, mice-threads.csv, etc.), then brief (scene-briefs.csv).

### Prompt Building

Each pass prompt includes:

1. The dimension being fixed — name, current score, target score, specific rationale from proposals.
2. Full current CSV data for affected scenes (same format as existing upstream revision prompts).
3. Relevant registries (values.csv, mice-threads.csv, characters.csv, locations.csv, etc.) so Claude uses canonical IDs.
4. Instructions to output pipe-delimited CSV in fenced blocks (`intent-csv`, `scenes-csv`, `briefs-csv`).

For **registry** fix_location (e.g., thematic concentration requiring values.csv consolidation):
- Prompt shows the full registry file and asks Claude to consolidate/merge entries.
- Output is a replacement registry CSV block.
- A second sub-step normalizes all scene-intent references to use the updated canonical values.

### CSV Application

Uses the existing `parse_stage_response` + `csv_block_to_rows` + `_read_csv_as_map` / `_write_csv` machinery from the revise script's upstream pass handling.

**Key difference from regular upstream revise:** The re-draft step after CSV application is skipped entirely. Scene markdown files are never read or written.

### Re-validation & Output

After all passes complete:

1. Re-run structural scoring (same as `storyforge-validate --structural --no-schema`).
2. Print before/after comparison for each dimension: old score, new score, delta.
3. Mark completed proposals as `completed` in `structural-proposals.csv`. If a dimension didn't improve, its proposals stay `pending`.
4. Commit all CSV changes and push.

### Dry Run

`storyforge-revise --structural --dry-run` prints the generated plan and prompts without invoking Claude, same as regular revise dry-run.

## Files Modified

- `scripts/storyforge-revise` — add `--structural` flag, plan generation from proposals, skip re-draft for structural passes.

## Interaction with Other Flags

- `--structural --dry-run` — works as expected.
- `--structural --interactive` — works; user can supervise each CSV pass.
- `--structural --coaching` — works; adjusts prompt posture.
- `--structural --polish` — error; mutually exclusive modes.
- `--structural` with existing `revision-plan.csv` — overwrites with the auto-generated structural plan (same as `--polish` behavior).
