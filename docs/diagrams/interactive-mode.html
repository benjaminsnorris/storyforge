Interactive Mode

Watch Claude work and intervene at any point — switching between supervised collaboration and autonomous autopilot mid-task

./storyforge write --interactive

Execution Modes

Two ways to work

Supervised

Interactive

Author collaborates with Claude step-by-step, giving feedback and redirecting as needed.

  • Banner displayed with instructions
  • System prompt rules appended
  • Claude completes one work unit, then waits
  • Author can redirect mid-task
Autonomous

Autopilot

Claude works autonomously through remaining tasks. Output streams to log files.

  • Progress monitor shows current status
  • Output streamed to log files
  • Brief pause between steps to allow rejoin
  • Activated via trigger phrases

Architecture

Core components

.
State Management

State File

A single file controls the current mode. Its presence means interactive mode is active.

working/.interactive
  • Created when --interactive flag is passed
  • Deleted on trigger phrase or script end
  • Cleaned up at script start and exit
i
Rejoin

Rejoin Mechanism

Between autopilot steps, a brief window lets the author jump back into interactive mode.

offer_interactive(): "Press 'i' for interactive, or wait 5s..."
  • Uses read -t $timeout -n 1 for keypress
  • Configurable via STORYFORGE_REJOIN_TIMEOUT
  • Default timeout: 5 seconds
B
Display

Interactive Banner

A boxed banner (60 chars wide) displayed at the start of each interactive step.

show_interactive_banner(subtitle, mode)
  • "multi" mode: shows autopilot escape option
  • "single" mode: no autopilot option shown
  • Instructions: watch, give feedback, redirect
$
Prompt Injection

System Prompt

Invisible rules appended via --append-system-prompt to constrain Claude's behavior.

build_interactive_system_prompt(dir, work_unit)
  • Complete THIS work unit only
  • Don't proceed to next step
  • Wait for user before continuing
  • Listen for trigger phrases

Compatibility

Scripts supporting interactive mode

Script Work Unit Interactive Scope
storyforge-write scene Scene-by-scene drafting with per-scene interaction
storyforge-revise pass Pass-by-pass revision with per-pass interaction
storyforge-evaluate synthesis Synthesis step only (evaluators run in parallel)
storyforge-assemble review Review phase only

*

Interactive mode is opt-in per invocation. Without the --interactive flag, all scripts run in fully autonomous headless mode. The working/.interactive state file is always cleaned up at script start and exit to prevent stale state.
