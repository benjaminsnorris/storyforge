Routing Layer

The Hub

/storyforge:forge reads project state from storyforge.yaml, CLAUDE.md, and artifact directories, then operates in one of three modes:

  • Directed — Author has a specific request; hub routes to the right skill.
  • Guided — Author says "surprise me"; hub delegates to the recommend skill for the highest-value next action.
  • Status — Author wants a summary of project state.
Execution Layer

Interactive Skills

Author collaborates with Claude in real time.

  • /storyforge:develop World, character, story, and timeline development.
  • /storyforge:voice Voice and style guide discovery.
  • /storyforge:scenes Scene index design and management.
  • /storyforge:plan-revision Plan revision passes from evaluation feedback.
  • /storyforge:recommend Assess project state, recommend the highest-value next action.
  • /storyforge:review Review revision results, map findings to changes.
  • /storyforge:title Title and subtitle development, refinement, assessment.
  • /storyforge:press-kit Blurbs, jacket copy, author bio, social media, marketing materials.
  • /storyforge:cover Cover design: SVG artwork or AI-generated illustrations.
Execution Layer

Autonomous Scripts

Claude executes independently without author interaction.

  • ./storyforge write Draft scenes using reference documents and the voice guide.
  • ./storyforge evaluate Run a 6-persona evaluation panel against the manuscript.
  • ./storyforge revise Execute the revision pipeline from evaluation results.
State Layer

Project State

All project state is file-based and version-controlled. The hub reads state before every routing decision.
