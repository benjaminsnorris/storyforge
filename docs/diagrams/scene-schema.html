Storyforge

The Scene Schema

Scenes are the atomic unit of a novel.

Every scene in Storyforge is a structured data object with rich metadata that connects it to the rest of the project. This schema enables autonomous drafting, evaluation, and revision by giving each scene enough context to be written and assessed independently. Each scene carries its own identity, narrative purpose, structural position, and connections to the broader manuscript.

The Schema
Identity
id string Unique identifier (e.g., act1-sc01)
title string Evocative scene title for reference
status string pending | drafted | revised | cut | merged
Narrative
pov string POV character's full name
setting string Where the scene takes place
characters list All characters present or referenced
function string Why this scene exists -- specific, not vague
emotional_arc string Emotional journey within the scene
Structure
part number Which act/part this scene belongs to
type string character | plot | world | action | transition
target_words number Target word count for drafting
timeline_position string Position in chronology (day, date, relative)
Connections
threads list Story threads touched (must match continuity tracker)
motifs list Recurring elements appearing in this scene
Scene Types
character
Character development, relationship dynamics, internal change
plot
Advances external story, reveals information, creates consequences
world
Establishes setting, rules, culture, atmosphere
action
Physical conflict, chase, confrontation, high-stakes moment
transition
Bridges time/space, repositions characters, modulates pacing

Most scenes blend types -- pick the dominant one.
