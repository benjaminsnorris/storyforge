Storyforge

Production & Publishing

Two paths to readers: publish to bookshelf for web reading, or produce epub/PDF for distribution.

Publish

/storyforge:publish

The one-stop "share this book" command. Assembles the web book, generates the dashboard, pushes everything to Bookshelf.

  • → Assemble web chapters
  • → Generate dashboard
  • → Push to Supabase
  • → Copy dashboard + cover
  • → Commit in bookshelf repo

Produce

/storyforge:produce

For formats beyond the web reader. Chapter mapping, production settings, format-specific output.

  • → Map scenes to chapters
  • → Configure front/back matter
  • → Select genre CSS preset
  • → Generate epub / PDF / HTML

The Publish Flow

1

Assemble Web Book

Scenes mapped to chapters via chapter-map.csv. HTML output with scene breaks, chapter headings, and reader navigation.

2

Generate Dashboard

Multi-page visualization: overview (spine, POV, rhythm), structure (threads, characters, timeline), scores (craft, fidelity, genre).

3

Push Content

Chapters and scenes upserted to Supabase. Existing highlights are re-anchored when content changes. Reading progress preserved.

4

Copy Assets

Dashboard and cover image copied to bookshelf public directory. Database updated with cover URL.

Format Options

The produce skill handles non-web formats with genre-appropriate styling.

Epub

Standard ebook format. Genre CSS presets: literary fiction, thriller, romance, fantasy, science fiction.

PDF

Print-ready output via pandoc + LaTeX. Configurable margins, fonts, and page size.

Web

Hostable static site with chapter navigation, scene breaks, and responsive design.
